# 猫神牧场 Demo 脚本使用说明

本文档用于维护 `Assets/Game/Scripts` 下的脚本。内容包括脚本功能、可在 Inspector 修改的参数、可被其他脚本调用的函数，以及脚本之间的调用关系。

## 项目运行主流程

1. 场景中手动放置 `Camera`、`EventSystem`、`RanchManager`、`RanchMap`、Canvas/UI、地块精灵等对象。
2. `RanchManager.Start()` 自动查找或使用 Inspector 中绑定的 `RanchMap`。
3. `RanchManager` 查找带 `SceneGridCellMarker` 的地块 SpriteRenderer，把它们交给 `RanchMap.Initialize(...)`。
4. `RanchMap` 读取每个地块的 `SceneGridCellMarker.GridCoords`，为地块添加或复用 `MapCell`，建立坐标到地块的字典。
5. `RanchManager` 从 `offerPool` 随机抽取初始动物，放到随机空地块。
6. `RanchUIController.Start()` 查找 `RanchManager`，绑定 HUD、抽卡面板，并订阅 `RanchManager.StateChanged`。
7. 玩家点击地块时，`MapCell.OnMouseDown()` 调用 `RanchManager.SelectCell(...)`。
8. 玩家点击结算按钮时，`RanchHUD` 调用 `RanchManager.NextDay()`。
9. `RanchManager.NextDay()` 按地图扫描顺序触发能力，结算基础收益，进入抽卡选择或下一天。

## 目录说明

- `Abilities`：动物能力数据、目标解析、效果注册、具体能力效果。
- `Animals`：动物运行时对象和动物数据资源定义。
- `Ranch`：牧场管理、地图、地块、动物视图、抽卡。
- `UI`：HUD 和动物选择界面。
- `Testing`：运行时测试面板。
- `Effects`：视觉动效。
- `Editor`：Unity 编辑器菜单工具。

## Abilities

### AbilityData.cs

功能：
- 定义能力配置资源，继承 `ScriptableObject`。
- 能力资源位于 `Assets/Game/Data/Abilities/Animals`。
- `AnimalData` 通过 `ability` 字段引用它。

可修改参数：
- `id`：能力 ID，也可作为默认 `EffectScriptId`。
- `abilityType`：能力类型。目前代码没有读取，属于预留字段。
- `priority`：能力优先级。目前代码没有排序使用，属于预留字段。
- `impactType`：影响范围，由 `AbilityTargetResolver` 读取。支持 `Self`、`Adjacent`、`Left`、`Right`、`Up/Upper/UpperAdjacent`、`Down/Lower/LowerAdjacent`、`All/Global/Field`。
- `stackable`：效果是否可叠加。目前 `AlpacaAbilityEffect` 使用。
- `desc`：能力描述，显示在选中动物信息里。
- `triggerType`：触发时机。当前常用值：`SettlementPrepare`、`DayStart`、`DayEnd`、`Moved`。
- `triggerChancePercent`：触发概率，0 到 100。
- `triggerLimit`：触发次数限制，0 表示不限制。
- `effectType`：效果类型。部分 Effect 会读取，如猪和羊要求 `SellSelf`。
- `effectScriptId`：效果脚本 ID，必须能在 `AbilityEffectRegistry` 找到。
- `effectParams`：通用效果参数，见 `AbilityEffectParams`。
- `subAbilities`：子能力列表。目前只用于 UI 描述拼接，实际不会自动执行。

可调用属性：
- `Id`、`AbilityType`、`Priority`、`ImpactType`、`Stackable`、`Desc`、`TriggerType`、`TriggerChancePercent`、`TriggerLimit`、`EffectType`、`EffectScriptId`、`EffectParams`、`SubAbilities`。

调用关系：
- 被 `AnimalData.Ability` 引用。
- 被 `AnimalAbilityFactory.Create(...)` 用于创建 `ConfiguredAnimalAbility`。
- 被 `ConfiguredAnimalAbility.TryExecute(...)` 判断触发条件并查找效果。
- 被 `AbilityTargetResolver.Resolve(...)` 解析作用目标。
- 被各个 `IAbilityEffect.Execute(...)` 读取参数。

维护注意：
- 新建能力后，如果 `effectScriptId` 没有注册，运行时不会生效。
- 当前 `CalfAbilityEffect`、`CowAbilityEffect`、`GoatAbilityEffect`、`LambAbilityEffect` 没有对应类和注册。

### AbilityEffectParams.cs

功能：
- 能力效果的通用参数容器，序列化在 `AbilityData.effectParams` 中。

可修改参数：
- `money`：通用数值。可能表示金币、每步金币、倍率、基础收益增加等，具体含义由 Effect 决定。
- `delayDays`：延迟天数。`ConfiguredAnimalAbility` 会用它判断 `Owner.AgeDays < delayDays` 时不触发。
- `durationDays`：持续天数。目前没有任何 Effect 使用，属于预留字段。
- `transformChancePercent`：转化概率。`BoarAbilityEffect` 使用。
- `type`：参数类型，如 `Flat`、`Multiplier`。目前没有统一读取，属于预留字段。
- `target`：目标筛选，`AbilityTargetResolver` 使用。支持 `Self`、`Other`。
- `targetFamily`：目标家族筛选，`AbilityTargetResolver` 使用。`None` 表示不限制。

调用关系：
- 被 `AbilityData` 持有。
- 被 `ConfiguredAnimalAbility` 读取 `delayDays`。
- 被 `AbilityTargetResolver` 读取 `target`、`targetFamily`。
- 被具体能力效果读取 `money`、`transformChancePercent` 等。

### AnimalAbilityContext.cs

功能：
- 能力执行上下文。
- 把 `RanchManager` 和当前能力拥有者 `Owner` 一起传给 Effect。

可调用成员：
- 构造函数 `AnimalAbilityContext(RanchManager ranchManager, Animal owner)`。
- `RanchManager`：当前牧场管理器。
- `Owner`：执行能力的动物。

调用关系：
- `RanchManager.ExecuteAbilityWithReport(...)` 创建它。
- `ConfiguredAnimalAbility.TryExecute(...)` 和所有 `IAbilityEffect` 使用它。

### IAnimalAbility.cs

功能：
- 动物能力运行时接口。

可调用成员：
- `Name`：能力名称。目前外部没有使用。
- `Priority`：能力优先级。目前外部没有使用。
- `TryExecute(AnimalAbilityContext context, string triggerType)`：尝试按触发类型执行能力。

调用关系：
- `Animal.Ability` 持有该接口。
- `ConfiguredAnimalAbility` 实现该接口。
- `RanchManager` 调用 `Animal.Ability.TryExecute(...)`。

### AnimalAbilityFactory.cs

功能：
- 根据 `AnimalData` 创建运行时能力对象。

可调用函数：
- `Create(AnimalData data)`：如果动物数据或能力为空，返回 `null`；否则返回 `ConfiguredAnimalAbility`。

调用关系：
- `Animal` 构造函数调用它。
- 依赖 `AnimalData.Ability`。

### ConfiguredAnimalAbility.cs

功能：
- 按 `AbilityData` 执行能力。
- 负责触发类型、触发次数、延迟天数、概率、效果注册查找、目标解析。

可调用成员：
- 构造函数 `ConfiguredAnimalAbility(AbilityData abilityData)`。
- `Name`：返回能力 ID。
- `Priority`：返回能力优先级。
- `TryExecute(AnimalAbilityContext context, string triggerType)`：满足条件时执行能力。

执行顺序：
1. 检查配置、Owner、RanchManager。
2. 比较 `config.TriggerType` 和传入 `triggerType`。
3. 检查 `TriggerLimit`。
4. 检查 `EffectParams.delayDays`。
5. 按 `TriggerChancePercent` 判定概率。
6. 用 `AbilityEffectRegistry.TryGet(...)` 找效果。
7. 用 `AbilityTargetResolver.Resolve(...)` 找目标。
8. 调用 `effect.Execute(...)`。
9. 如果效果返回 `true`，增加 `triggerCount`。

调用关系：
- 由 `AnimalAbilityFactory` 创建。
- 调用 `AbilityEffectRegistry`、`AbilityTargetResolver`、具体 `IAbilityEffect`。
- 被 `RanchManager` 间接调用。

### AbilityEffectRegistry.cs

功能：
- 注册 `effectScriptId` 到具体 `IAbilityEffect` 实例。

已注册 ID：
- `PigAbilityEffect`
- `BoarAbilityEffect`
- `HorseAbilityEffect`
- `SheepAbilityEffect`
- `ZebraAbilityEffect`
- `GazelleAbilityEffect`
- `DonkeyAbilityEffect`
- `AlpacaAbilityEffect`

可调用函数：
- `TryGet(string effectScriptId, out IAbilityEffect effect)`：查找效果。

调用关系：
- 被 `ConfiguredAnimalAbility.TryExecute(...)` 调用。

扩展方式：
1. 新建一个类实现 `IAbilityEffect`。
2. 在 `AbilityEffectRegistry.Effects` 字典里添加 `{ "YourEffectId", new YourEffect() }`。
3. 在对应 `AbilityData.asset` 中设置 `effectScriptId: YourEffectId`。

### AbilityTargetResolver.cs

功能：
- 根据能力配置的 `impactType` 和 `effectParams.target/targetFamily` 解析目标动物列表。

可调用函数：
- `Resolve(AnimalAbilityContext context, AbilityData abilityData)`：返回目标动物列表。

范围规则：
- `Self`：当前动物所在格。
- `Adjacent`：左右、上邻、下邻。
- `Left`：左侧一格。
- `Right`：右侧一格。
- `Up/Upper/UpperAdjacent`：上方相邻格。
- `Down/Lower/LowerAdjacent`：下方相邻格。
- `All/Global/Field`：全地图。

筛选规则：
- `target = Self`：只保留自己。
- `target = Other`：排除自己。
- `targetFamily != None`：只保留同家族目标。

调用关系：
- 被 `ConfiguredAnimalAbility.TryExecute(...)` 调用。
- 依赖 `RanchMap` 的邻居查询函数。

### IAbilityEffect.cs

功能：
- 能力效果接口。

可调用函数：
- `Execute(AnimalAbilityContext context, AbilityData abilityData, IReadOnlyList<Animal> targets)`：执行效果，返回是否成功应用。

调用关系：
- 所有具体能力效果实现它。
- `ConfiguredAnimalAbility` 调用它。

### PigAbilityEffect.cs

功能：
- 猪的能力效果：满足条件后卖掉自己并获得金币。

执行条件：
- `context.Owner`、`context.RanchManager`、`abilityData` 不为空。
- `abilityData.EffectType` 必须是 `SellSelf`。

使用参数：
- `effectParams.money`：卖出获得的金币。

调用关系：
- 调用 `RanchManager.RemoveAnimal(...)` 删除自己。
- 调用 `RanchManager.AddMoney(...)` 加钱。
- 由 `AbilityEffectRegistry` 注册为 `PigAbilityEffect`。

### SheepAbilityEffect.cs

功能：
- 羊的能力效果：满足条件后卖掉自己并获得金币。

执行条件和调用关系：
- 与 `PigAbilityEffect` 基本相同。
- 由 `AbilityEffectRegistry` 注册为 `SheepAbilityEffect`。

### BoarAbilityEffect.cs

功能：
- 野猪能力效果：检查野猪数量，并可能把相邻猪转化为野猪。

使用参数：
- `effectParams.money`：当场上野猪数量大于等于 2 时获得的金币。
- `effectParams.transformChancePercent`：猪转化为野猪的概率。

执行逻辑：
1. 调用 `RanchManager.CountAnimalsById("Boar")`。
2. 如果野猪数量大于等于 2，调用 `RanchManager.AddMoney(...)`。
3. 从 `targets` 中筛选 `Data.Id == "Pig"` 的目标。
4. 按概率随机选择一只猪。
5. 调用 `RanchManager.ReplaceAnimal(selectedPig, context.Owner.Data)` 把猪替换成野猪。

调用关系：
- 被 `AbilityEffectRegistry` 注册为 `BoarAbilityEffect`。
- 依赖 `AbilityTargetResolver` 提供相邻目标。

### HorseAbilityEffect.cs

功能：
- 马的能力效果：沿当前行向右移动到最远空格，并按移动步数获得金币。

使用参数：
- `effectParams.money`：每移动一格获得的金币。

执行逻辑：
1. 从当前 `Owner.Coords` 开始向右查找连续空地。
2. 如果能移动，调用 `RanchManager.TryMoveAnimal(...)`。
3. 按 `移动步数 * money` 加钱。

调用关系：
- 使用 `RanchMap.Width`、`RanchMap.TryGetCell(...)`。
- 调用 `RanchManager.TryMoveAnimal(...)`、`RanchManager.AddMoney(...)`。
- 被 `AbilityEffectRegistry` 注册为 `HorseAbilityEffect`。

注意：
- `TryMoveAnimal(...)` 成功后会触发该动物的 `Moved` 能力。如果以后有连锁移动能力，要注意递归或连锁次数。

### GazelleAbilityEffect.cs

功能：
- 瞪羚能力效果：与上方第一个有动物的相邻格交换，并获得金币。

使用参数：
- `effectParams.money`：交换成功后获得的金币。

执行逻辑：
1. 调用 `RanchMap.GetFirstUpperNeighborWithAnimal(...)`。
2. 调用 `RanchManager.TrySwapAnimals(...)`。
3. 成功后按参数加钱。

调用关系：
- 使用 `RanchManager.Map`。
- 调用 `RanchManager.TrySwapAnimals(...)`、`RanchManager.AddMoney(...)`。
- 被 `AbilityEffectRegistry` 注册为 `GazelleAbilityEffect`。

### DonkeyAbilityEffect.cs

功能：
- 驴的能力效果：如果场上存在马或斑马，则获得金币。

使用参数：
- `effectParams.money`：奖励金币。

执行逻辑：
- 调用 `RanchManager.HasAnimalById("Horse")` 和 `HasAnimalById("Zebra")`。
- 条件满足后调用 `RanchManager.AddMoney(...)`。

调用关系：
- 被 `AbilityEffectRegistry` 注册为 `DonkeyAbilityEffect`。

### ZebraAbilityEffect.cs

功能：
- 斑马能力效果：给自己永久增加基础收益。

使用参数：
- `effectParams.money`：增加的基础收益值。

执行逻辑：
- 调用 `context.Owner.AddBaseMoneyBonus(...)`。

调用关系：
- 被 `AbilityEffectRegistry` 注册为 `ZebraAbilityEffect`。

注意：
- 当前加成是永久叠加的，`ResetSettlementModifiers()` 不会清除 `baseMoneyBonus`。

### AlpacaAbilityEffect.cs

功能：
- 羊驼能力效果：给目标添加额外金币倍率。

使用参数：
- `effectParams.money`：倍率值。大于 1 才会生效。
- `abilityData.Stackable`：控制倍率是否乘法叠加。

执行逻辑：
- 遍历 `targets`，调用 `target.AddExtraMoneyMultiplier(multiplier, stackable)`。

调用关系：
- 被 `AbilityEffectRegistry` 注册为 `AlpacaAbilityEffect`。
- 影响 `RanchManager.AddMoney(...)` 中通过 `activeExtraMoneyOwner` 结算的额外金币。

## Animals

### AnimalData.cs

功能：
- 动物配置资源，继承 `ScriptableObject`。
- 动物资源位于 `Assets/Game/Data/Animals/Hoofed`。

可修改参数：
- `id`：动物 ID，能力代码中常用它判断动物种类，如 `Pig`、`Boar`、`Horse`。
- `animalName`：显示名称。为空时使用资源名。
- `family`：动物家族，如 `Hoofed`。
- `rarity`：稀有度，0 到 4。
- `baseMoney`：基础收益。
- `ability`：引用 `AbilityData`。
- `description`：描述文本。
- `icon`：动物图标。
- `iconScale`：图标缩放倍率。

可调用属性：
- `Id`、`Name`、`Family`、`Rarity`、`BaseMoney`、`Ability`、`Description`、`Icon`、`IconScale`。
- `AnimalId`：`Id` 的别名，目前没有使用。
- `DisplayName`：`Name` 的别名，UI 和日志常用。
- `AbilityDescription`：能力描述或动物描述，目前没有使用。

可调用函数：
- `Initialize(...)`：运行时或编辑器代码可批量设置字段。目前项目没有调用。

调用关系：
- `Animal` 持有它。
- `RanchManager` 的 `offerPool` 和初始动物抽取使用它。
- `AnimalOfferRoller` 使用 `Rarity`、`DisplayName`。
- `AnimalView` 使用 `Icon`、`IconScale`。
- 能力效果使用 `Id`、`Family`。

### Animal.cs

功能：
- 动物的运行时实例。
- 保存动物数据、坐标、年龄、运行时能力、临时收益修正。

可调用属性：
- `Data`：动物配置。
- `Ability`：运行时能力。
- `Coords`：当前地图坐标。
- `AgeDays`：已结算天数。
- `DisplayName`：显示名称。
- `BaseMoney`：配置基础收益加永久基础收益加成。

可调用函数：
- `Animal(AnimalData data, Vector2Int coords)`：构造动物实例。
- `SetCoords(Vector2Int coords)`：更新坐标。
- `ResolveBaseMoney(RanchMap map)`：结算基础收益，并推进年龄和临时倍率天数。
- `AddIncomeMultiplierBonus(int bonus, int durationDays)`：添加临时基础收益倍率加成。目前没有调用方。
- `AddExtraMoneyMultiplier(int multiplier, bool stackable)`：添加额外金币倍率。
- `ResolveExtraMoney(int amount)`：计算额外金币倍率后的金额。
- `ResetSettlementModifiers()`：重置每日临时额外金币倍率。
- `AddBaseMoneyBonus(int bonus)`：永久增加基础收益。
- `ResolveDayStartAbility(...)`、`ResolveDayEndAbility(...)`：旧式能力入口，目前没有调用。

调用关系：
- 由 `RanchManager` 创建、移动、删除。
- 由 `MapCell` 持有和显示。
- 由能力系统读取或修改。

维护注意：
- `ResolveBaseMoney(RanchMap map)` 的 `map` 参数当前没有被使用。
- `AddIncomeMultiplierBonus(...)` 对基础收益生效；`AddExtraMoneyMultiplier(...)` 对能力额外加钱生效，两者不是一回事。

## Ranch

### RanchManager.cs

功能：
- 牧场主控制器，管理天数、金币、地图、动物列表、选中地块、每日结算、抽卡、测试模式。

可修改参数：
- `mapWidth`：地图宽度。
- `mapHeight`：地图高度。
- `day`：初始天数。
- `money`：初始金币。
- `ranchMap`：场景中的 `RanchMap`。可不填，运行时会查找。
- `offerPool`：可抽取动物池，也用于随机初始动物。
- `offerRoller`：抽卡器。可不填，运行时从同物体查找。
- `animalViewPrefab`：动物显示 Prefab。
- `fallbackAnimalSprite`：动物没有图标时使用的兜底图。
- `randomStartingAnimalCount`：随机初始动物数量。

可调用事件：
- `StateChanged`：状态变化时触发，UI 订阅它刷新。

可调用属性：
- `Day`、`Money`、`Map`、`SelectedCell`、`IsWaitingForOfferSelection`、`IsWaitingToEnterNextDay`、`IsTestMode`、`CurrentOffers`、`LastSettlementReport`。

可调用函数：
- `Initialize(RanchMap map, IReadOnlyList<AnimalData> startingAnimals, Sprite tileSprite, Sprite animalSprite, IReadOnlyList<SpriteRenderer> sceneTiles = null)`：手动初始化入口。
- `SelectCell(MapCell cell)`：选中地块。
- `NextDay()`：结算当天或进入下一天。
- `AddMoney(int amount)`：以当前能力拥有者为来源添加额外金币。
- `AddExtraMoney(Animal source, int amount)`：按指定动物的额外金币倍率添加金币。
- `TryMoveAnimal(Animal animal, Vector2Int targetCoords)`：移动动物，成功后触发 `Moved` 能力。
- `TrySwapAnimals(Animal first, Animal second)`：交换动物，成功后分别触发 `Moved` 能力。
- `RemoveAnimal(Animal animal)`：从地图和列表移除动物。
- `ReplaceAnimal(Animal oldAnimal, AnimalData newAnimalData)`：用新动物数据替换旧动物。
- `TrySetAnimalAt(Vector2Int coords, AnimalData animalData)`：测试/工具用，在指定格设置动物。
- `TryClearAnimalAt(Vector2Int coords)`：测试/工具用，清空指定格动物。
- `ClearAllAnimals()`：清空所有动物。
- `EnterTestMode()`：进入测试模式，清空动物，关闭抽卡等待。
- `ExitTestMode()`：退出测试模式。
- `SelectOffer(int index)`：选择抽卡结果，并添加到随机空地。
- `CountAnimalsById(string animalId)`：统计某 ID 动物数量。
- `HasAnimalById(string animalId)`：判断是否存在某 ID 动物。
- `GetSelectedCellText()`：生成 UI 选中格说明。

主要内部流程：
- `Start()` -> `InitializeFromScene()`。
- `InitializeFromScene()` 查找 `RanchMap`、抽初始动物、查找地块。
- `ResolveDailySettlement()` 按顺序触发 `SettlementPrepare`、`DayStart`、`DayEnd`，再结算基础收益。
- `ExecuteAbilityWithReport(...)` 包装能力执行，并记录能力造成的金币变化。
- `RollOffers(...)` 调用 `AnimalOfferRoller` 或备用随机抽卡。
- `RandomizeAnimalPositions()` 进入下一天时随机重排动物位置。

调用关系：
- 调用 `RanchMap` 初始化、查询、移动、交换、删除。
- 创建和持有 `Animal`。
- 创建 `AnimalAbilityContext` 并调用 `Animal.Ability.TryExecute(...)`。
- 被 `RanchUIController`、`RanchHUD`、`AnimalOfferPanel`、`RanchTestController`、能力效果调用。

维护注意：
- `NextDay()` 有两个阶段：第一次点击结算，第二次点击进入下一天。

## 新增动物流程

1. 在 `Assets/Game/Data/Animals/...` 创建或复制一个 `AnimalData`。
2. 设置 `id`、`animalName`、`family`、`rarity`、`baseMoney`、`icon`、`iconScale`。
3. 如果动物有能力，创建或复制一个 `AbilityData` 并赋给 `AnimalData.ability`。
4. 把动物加入 `RanchManager.offerPool` 或 `AnimalOfferRoller.offerPool`。
5. 如果能力用新效果，新增 `IAbilityEffect` 实现，并注册到 `AbilityEffectRegistry`。

## 新增能力效果流程

1. 在 `Assets/Game/Scripts/Abilities` 新建 `YourAbilityEffect.cs`。
2. 实现 `IAbilityEffect.Execute(...)`。
3. 在 `AbilityEffectRegistry.Effects` 添加 `{ "YourAbilityEffect", new YourAbilityEffect() }`。
4. 在对应 `AbilityData.asset` 设置 `effectScriptId: YourAbilityEffect`。
5. 根据需要设置 `triggerType`、`impactType`、`effectParams`。

## 常见排查

- 动物不显示：检查 `RanchManager.animalViewPrefab`、动物 `icon`、`MapCell` 是否成功创建 `AnimalView`。
- 地图不能点击：检查地块是否有 `BoxCollider2D`，场景是否有 `EventSystem`，鼠标是否点在 UI 上。
- 地图初始化失败：检查每个地块是否有唯一且完整的 `SceneGridCellMarker.gridCoords`。
- 能力不触发：检查 `triggerType` 是否匹配、`delayDays` 是否满足、概率是否通过、`effectScriptId` 是否注册。
- 抽卡不出现：检查 `offerPool` 是否为空，是否正在 `testMode`，是否已经处于等待下一天状态。
